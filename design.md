# Дизайн-документ: Dyna-Q Food Delivery

## 1. Огляд проєкту

**Назва:** Dyna-Q Food Delivery Simulator
**Тип:** Браузерний додаток (Single Page Application)
**Стек:** Чистий HTML + CSS + JavaScript (zero dependencies)
**Мета:** Дослідницький інструмент для вивчення алгоритму Dyna-Q на прикладі задачі доставки їжі по місту. Дозволяє експериментувати з гіперпараметрами та спостерігати їх вплив на процес навчання.

---

## 2. Предметна область

### Сценарій
Кур'єр працює у місті, представленому сіткою 10×10. На карті розташовані:
- **Ресторани** (точки забору замовлень) — 2-3 фіксовані позиції
- **Клієнти** (точки доставки) — генеруються динамічно після забору замовлення
- **Стіни** (будівлі) — непрохідні клітинки, формують вуличну мережу
- **Кур'єр** (агент) — рухається по вільних клітинках сітки

### Логіка доставки
1. На початку симуляції кур'єр з'являється на випадковій вільній клітинці
2. Їде до ресторану, щоб забрати замовлення
3. Після забору замовлення з'являється точка доставки (клієнт)
4. Кур'єр доставляє замовлення клієнту
5. Епізод завершується після доставки (або при досягненні ліміту кроків)
6. Новий епізод — кур'єр залишається на поточній позиції, генерується новий клієнт/ціль

## 3. Формалізація MDP

### Простір станів (State Space)
Стан визначається кортежем:
```
S = (x, y, has_order)
```
- `(x, y)` — позиція кур'єра на сітці [0..9, 0..9]
- `has_order` — чи несе кур'єр замовлення (0 або 1)

**Загальна кількість станів:** 10 × 10 × 2 = 200

### Простір дій (Action Space)
4 дискретні дії:
```
A = { UP, DOWN, LEFT, RIGHT }
```
- Рух на одну клітинку у відповідному напрямку
- Якщо дія веде в стіну або за межі сітки — агент залишається на місці

### Функція переходів (Transition Function)
- **Детерміновані переходи:** агент завжди рухається у вибраному напрямку (якщо клітинка вільна)
- Спрощена модель без стохастичності — фокус на демонстрації planning-компоненти Dyna-Q

### Функція винагороди (Reward Function)
| Подія                                    | Винагорода |
|------------------------------------------|-----------|
| Забрав замовлення (дійшов до ресторану)  | +5        |
| Доставив замовлення (дійшов до клієнта)  | +20       |
| Кожний крок                              | -1        |
| Спроба піти в стіну / за межі сітки     | -1        |

### Коефіцієнт дисконтування
```
γ ∈ {0.5, 0.8, 0.9, 0.95, 0.99}   (вибирається через dropdown)
```

## 4. Алгоритм Dyna-Q

### Псевдокод
```
Initialize Q(s, a) = 0 for all s, a
Initialize Model(s, a) = {} (порожня модель)

For each episode:
    s = current state (agent stays where it is)
    generate new customer position
    While not terminal:
        // 1. Вибір дії (ε-greedy)
        a = ε-greedy(Q, s, ε)

        // 2. Виконання дії в реальному середовищі
        (s', r, done) = Environment.step(s, a)

        // 3. Direct RL (Q-learning update)
        Q(s,a) += α * [r + γ * max_a' Q(s',a') - Q(s,a)]

        // 4. Оновлення моделі
        Model(s, a) = (s', r)

        // 5. Planning (n кроків Dyna)
        For i = 1 to n:
            s_sim = random previously visited state
            a_sim = random previously taken action in s_sim
            (s'_sim, r_sim) = Model(s_sim, a_sim)
            Q(s_sim, a_sim) += α * [r_sim + γ * max_a' Q(s'_sim, a') - Q(s_sim, a_sim)]

        s = s'
```

### Гіперпараметри (слайдери)

| Параметр | Опис | Мін | Макс | Крок | За замовч. |
|----------|------|-----|------|------|-----------|
| `α` (learning rate) | Швидкість навчання | 0.01 | 0.5 | 0.01 | 0.1 |
| `γ` (discount factor) | Коефіцієнт дисконтування | 0.5 | 0.99 | 0.01 | 0.95 |
| `ε` (exploration rate) | Ймовірність випадкової дії | 0.01 | 0.5 | 0.01 | 0.1 |
| `n` (planning steps) | Кількість planning-кроків | 0 | 50 | 1 | 10 |
| `episodes` | Кількість епізодів навчання | 50 | 2000 | 50 | 500 |
| `max_steps` | Макс. кроків на епізод | 50 | 500 | 50 | 200 |

Кожен слайдер — HTML `<input type="range">` з відображенням поточного значення поруч.

> **Ключова дослідницька фішка:** при `n=0` алгоритм вироджується у звичайний Q-learning — можна порівняти ефект planning-кроків.

## 5. Архітектура додатку

### Структура файлів
```
index.html              — єдиний файл: розмітка + стилі + JavaScript
```

Весь код в одному HTML-файлі для максимальної простоти та портативності. JavaScript розбитий на логічні секції всередині `<script>`:

### Модулі (секції коду)

#### `Environment`
```
Відповідальність: карта міста, фізика, винагороди

- GRID_SIZE = 10
- grid[10][10]           — карта (EMPTY=0, WALL=1, RESTAURANT=2, CUSTOMER=3)
- agentPos {x, y}        — позиція кур'єра
- hasOrder               — чи несе замовлення
- restaurantPositions[]   — координати ресторанів (фіксовані)
- customerPos {x, y}     — координати клієнта (генерується)

- reset()                — скинути стан епізоду: hasOrder=false, згенерувати нового клієнта; позиція кур'єра НЕ змінюється
- resetFull()            — повний скид (при перезапуску): випадкова позиція кур'єра + reset()
- step(action)           — виконати дію → {nextState, reward, done}
- getState()             → string "x,y,hasOrder"
- generateCustomer()     — випадкова вільна клітинка для клієнта
- isValidMove(x, y)      → boolean
- buildDefaultMap()      — створити карту з стінами та ресторанами
```

#### `DynaQAgent`
```
Відповідальність: навчання, Q-таблиця, модель середовища

- Q = {}                 — Q-таблиця {stateKey: [q_up, q_down, q_left, q_right]}
- model = {}             — модель {stateKey_action: {nextState, reward}}
- visitedPairs = []      — список (state, action) для planning
- alpha, gamma, epsilon  — гіперпараметри
- planningSteps          — кількість Dyna-кроків

- chooseAction(state)    → action (ε-greedy)
- learn(s, a, r, s')     — Q-learning update
- updateModel(s, a, r, s') — зберегти перехід у модель
- planning()             — n кроків planning з моделі
- getQValues(state)      → [q0, q1, q2, q3]
- reset()                — очистити Q-таблицю та модель
```

#### `Renderer`
```
Відповідальність: відмальовка карти міста на Canvas

- canvas, ctx            — Canvas елемент та контекст
- cellSize               — розмір клітинки (px)

- render(env)            — повна перемальовка: сітка + сутності + агент
- drawGrid(grid)         — малювання клітинок (стіни, пусто)
- drawRestaurants(positions) — іконки ресторанів
- drawCustomer(pos)      — іконка клієнта
- drawAgent(pos, hasOrder)  — іконка кур'єра
- drawPath(path)         — пройдений шлях (опціонально)
```

#### `Chart`
```
Відповідальність: графік середньої винагороди

- canvas, ctx            — Canvas для графіка
- data = []              — масив винагород по епізодах
- windowSize             — розмір ковзного вікна (n останніх епізодів)

- addDataPoint(reward)   — додати точку
- render()               — перемалювати графік
- drawAxes()             — осі X (епізоди) та Y (середня винагорода)
- drawLine(data)         — лінія ковзного середнього
- clear()                — очистити дані
```

#### `App` (main)
```
Відповідальність: UI, цикл навчання, зв'язок модулів

- env                    — екземпляр Environment
- agent                  — екземпляр DynaQAgent
- renderer               — екземпляр Renderer
- chart                  — екземпляр Chart
- isRunning              — прапорець запуску
- currentEpisode         — лічильник епізодів
- episodeRewards[]       — масив винагород по епізодах

- init()                 — ініціалізація UI, обробники подій
- start()                — запустити цикл навчання
- reset()                — скинути все, зчитати нові гіперпараметри з dropdown
- runEpisode()           — один епізод навчання з візуалізацією
- updateStats()          — оновити статистику на екрані
- readHyperparams()      — зчитати значення зі слайдерів
```

### Поведінка UI

#### Слайдери гіперпараметрів
- HTML `<input type="range">` з відображенням поточного числового значення праворуч
- Плавне перетягування з кроком, зазначеним у таблиці
- Зміна значення слайдера **не впливає** на поточний запуск
- Нові значення застосовуються тільки після натискання "Перезапуск"

#### Кнопка "Перезапуск" (Reset)
При натисканні:
1. Зупиняє поточне навчання
2. Зчитує всі значення зі слайдерів
3. Створює нового агента з новими гіперпараметрами (Q-таблиця = 0)
4. Скидає середовище (кур'єр на випадкову позицію)
5. Очищає графік
6. Скидає лічильники (епізод, винагорода, доставки)
7. Автоматично запускає нове навчання

#### Графік: Середня винагорода за останні N епізодів
- **Вісь X:** номер епізоду (1, 2, 3, ...)
- **Вісь Y:** середня сумарна винагорода за останні 20 епізодів (ковзне середнє)
- Оновлюється після кожного завершеного епізоду
- Автоматичне масштабування осей
- Очищається при перезапуску

#### Статистика (текстова)
- Номер поточного епізоду / загальна кількість
- Сумарна винагорода поточного епізоду
- Загальна кількість успішних доставок

#### Швидкість навчання
- Навчання відбувається з візуалізацією кожного N-го кроку
- Баланс між наочністю та швидкістю
- Один "тік" = один крок агента у середовищі + оновлення Canvas

## 8. Сценарії дослідження

### Експеримент 1: Вплив planning steps (n)
1. Запустити з n=0 (чистий Q-learning), запам'ятати графік
2. Перезапустити з n=5, n=10, n=50
3. Порівняти швидкість збіжності та фінальну якість
4. **Очікування:** більше n → швидша збіжність, але повільніший один епізод

### Експеримент 2: Вплив exploration (ε)
1. Запустити з ε=0.01 (майже жадібний)
2. Перезапустити з ε=0.1, ε=0.3
3. **Очікування:** занадто мале ε → застрягає в субоптимальній політиці; занадто велике → повільна збіжність

### Експеримент 3: Вплив learning rate (α)
1. Порівняти α=0.01 vs α=0.1 vs α=0.5
2. **Очікування:** мале α → повільне, але стабільне; велике α → швидке, але нестабільне

### Експеримент 4: Вплив discount factor (γ)
1. Порівняти γ=0.5 vs γ=0.95 vs γ=0.99
2. **Очікування:** мале γ → "короткозорий" агент; велике γ → довгострокове планування

## 10. Технічні деталі

### Кодування стану
```javascript
function encodeState(x, y, hasOrder) {
    return `${x},${y},${hasOrder ? 1 : 0}`;
}
```

### Q-таблиця
```javascript
// Об'єкт, ключі = рядки стану, значення = масив [q_UP, q_DOWN, q_LEFT, q_RIGHT]
Q = {
    "3,7,0": [0.0, 0.0, 0.0, 0.0],
    "3,7,1": [-0.5, 1.2, 0.0, -0.1],
    ...
}
```

### Модель середовища
```javascript
// Об'єкт, ключі = "state|action", значення = {nextState, reward}
model = {
    "3,7,0|2": { nextState: "2,7,0", reward: -1 },
    ...
}
```

### Цикл навчання
```javascript
// Кожен тік = один крок агента
function tick() {
    if (episodeDone) {
        chart.addDataPoint(episodeReward);
        currentEpisode++;
        if (currentEpisode >= totalEpisodes) { stop(); return; }
        env.reset(); // кур'єр залишається на місці, генерується новий клієнт
        episodeReward = 0;
    }

    const state = env.getState();
    const action = agent.chooseAction(state);
    const { nextState, reward, done } = env.step(action);
    agent.learn(state, action, reward, nextState);
    agent.updateModel(state, action, reward, nextState);
    agent.planning();
    episodeReward += reward;
    episodeDone = done;

    renderer.render(env);
    updateStats();

    if (isRunning) setTimeout(tick, tickDelay);
}
```
